Upload Key Status

The Library service role key is managed securely on the server (Cloudflare environment variables) and is used only by the /api/library-storage function. It never reaches your browser — no manual entry, no sessionStorage.

Service key managed securely on the server — no manual entry needed
Required Cloudflare variables (one-time): LIBRARY_SUPABASE_URL, LIBRARY_SUPABASE_SERVICE_KEY, LIBRARY_ADMIN_PASSCODE (must equal this dashboard's passcode).

Alternative: Allow Uploads via Anon Key

If you prefer not to use the service role key in the browser, run this SQL in the Supabase SQL Editor to allow the anon role to upload to each bucket.

-- Run in Supabase SQL Editor for each bucket:
CREATE POLICY "Admin anon uploads" ON storage.objects
  FOR INSERT TO anon
  WITH CHECK (bucket_id IN ('library-covers','library-books','library-audio','library-videos'));

-- Also allow reading (required for public URL display):
CREATE POLICY "Admin anon reads" ON storage.objects
  FOR SELECT TO anon
  USING (bucket_id IN ('library-covers','library-books','library-audio','library-videos'));

-- Allow updates (x-upsert):
CREATE POLICY "Admin anon updates" ON storage.objects
  FOR UPDATE TO anon
  USING (bucket_id IN ('library-covers','library-books','library-audio','library-videos'));
